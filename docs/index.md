# Triplet Experiment Template

Ready-to-use software and template for running **triadic comparison experiments** (triplet tasks) online using [jsPsych](https://www.jspsych.org/) v7.

On each trial, participants view three images — one target on top, two choices below — and select the choice most similar to the target. The template includes attention checks, validation trials, progress tracking, and automated data export.

---

## Quick start

The repo is installed **once** and shared across all your studies. Each study lives in its own subdirectory under `my_studies/`.

```bash
# 1. Clone and install (once per machine)
git clone <repo-url>
cd triplet_experiment_template
npm install

# 2. Create a directory for your study
mkdir my_studies/my_new_study

# 3. Copy the template config into your study directory and edit it
cp experiment.yaml my_studies/my_new_study/experiment.yaml
#    — set stimuli_dir: assets/stimuli  (relative to your study directory)
#    — add your images to my_studies/my_new_study/assets/stimuli/
#    — set your credentials, trial counts, and UI text

# 4. Generate the experiment files
node scripts/generate_experiment.js my_studies/my_new_study

# 5. Preview locally
cd my_studies/my_new_study && python -m http.server 8000
```

Then open `http://localhost:8000`.

## Repository layout

```
├── experiment/            # Shared experiment template (HTML + JS) — do not edit
│   ├── index.html
│   ├── js/
│   │   └── utils.js       # Trial-generation utilities
│   └── assets/            # Placeholder assets for local testing only
├── experiment.yaml        # Template config — copy this into your study directory
├── my_studies/            # Your studies go here (contents are git-ignored)
│   └── my_new_study/      # One directory per study
│       ├── experiment.yaml
│       ├── assets/
│       │   ├── stimuli/   # Your stimulus images
│       │   └── consent/   # Your consent form
│       ├── index.html     # Copied from template by the generator
│       └── js/            # Generated by the generator
├── scripts/
│   └── generate_experiment.js  # Generator — run with a study path to build it
├── tests/                 # JavaScript unit tests (vitest)
└── docs/                  # This documentation site
```
